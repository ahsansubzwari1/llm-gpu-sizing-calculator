Model configuration
Model preset
Layers i The number of transformer layers in the model. Each layer requires its own Key and Value vectors stored in memory. More layers = larger KV cache.
KV heads i Number of Key/Value attention heads after Grouped Query Attention (GQA). GQA shares KV heads across multiple query heads, drastically reducing KV cache size. Most modern models use 8 KV heads.
Head dim i The dimensionality of each attention head's key/value vectors. Common values are 64 and 128. Larger values increase memory per token proportionally.
Params (B) i Total model parameter count in billions. Used to estimate model weight memory: params × bytes per precision. For MoE models, use the total parameter count — all weights must reside in VRAM even if only a subset are active per token.
Weight precision i Numerical format used to store model weights in GPU memory. Lower precision reduces memory but may affect output quality. BF16 = 2 bytes/param (full quality). FP8 = 1 byte/param (minimal quality loss). INT4 = 0.5 bytes/param (noticeable quality trade-off).
KV cache precision i Numerical format used to store the KV cache during inference. Modern serving frameworks like vLLM can quantize the KV cache independently from model weights, saving significant memory with minimal quality impact.
Concurrent users i Number of simultaneous inference sessions the GPU must support. Each active session requires its own KV cache allocation in VRAM. Higher concurrency multiplies KV cache memory linearly — this is often the dominant variable for large context workloads.
5
Context window (tokens) i Maximum number of tokens (roughly 0.75 words each) in a single session's context. The KV cache must store representations for every token in the active context. 128K tokens ≈ ~96,000 words, or a large codebase. Longer context = dramatically more memory per session.
128K
Avg context utilization i The average fraction of the maximum context window actually occupied at any moment across all sessions. Sessions build context gradually — early turns are short. 60% is a realistic default for agentic coding workloads. RAG workloads are typically lower (30–50%) since sessions are short-lived.
60%
Framework overhead (GB) i VRAM reserved by the serving framework (e.g., vLLM, TensorRT-LLM) for CUDA kernels, activation buffers, attention computation workspace, and runtime state. Typically 10–20 GB depending on framework and batch size. Default of 14 GB is appropriate for vLLM with moderate batch sizes.
